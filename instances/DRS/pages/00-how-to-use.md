# 0. How to Use This Playbook

> `Owner Platform Owner` · `Status implemented` · `Depends on —`

**Purpose** — orient a reader in one screen.

## The approach

Each page is one topic: a **codification strip** (owner, scope, dependencies) at the top, then the meat —
purpose, prose, any graphics, a **decisions table**, and supplementary tables. Read the prose and
diagrams; the decisions table is the scannable spine.

This is a **client instance**, compiled from the library against the assessment. The archetype
(see `../../../archetypes.md`) pre-fills every decision's default; you confirm or adjust per topic.

- **Options** in each decisions table show the A1/A2/A3 defaults plus **Other** — pick one or write your own.
- **Status:** `proposed` (not yet ratified) · `agreed` · `implemented`.
- **Agents** read this per `../../../AGENTS.md`: act on agreed/implemented, honour `constraints:`, ask when unknown.

## Where to start

**This instance is scoped front-end only** (see [../README.md](../README.md)): pages 07–10 plus
writeback. Back-end topics live in DRS's DevOps wiki and are not carried here.
